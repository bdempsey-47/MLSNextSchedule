### Youth Soccer Schedules Platform

**Complexity:** high | **Commits:** 47 | **Period:** 2026-02-26 to 2026-03-10

Built a comprehensive full-stack youth soccer scheduling and analytics platform called YSS (Youth Soccer Schedules), initially branded as MLSNext. The system ingests real-time match data from the Modular11 API, provides filtering and visualization of schedules, computes standings with advanced analytics, and displays momentum indices and strength of schedule metrics for teams across multiple leagues and age groups.


**Phases:**

1. Phase 1: Backend Foundation & Data Ingestion (Feb 26 - Feb 28): Established C# backend infrastructure with HTML parsing for Modular11 API, implemented score parsing and team association logic, created database schema with EF Core migrations, and initial data population.

2. Phase 2: Frontend & UI Development (Feb 27 - Mar 1): Built React frontend with modern sports-league aesthetic, implemented match filtering by season/age group/region, added team logos from CDN, venue mapping, and calendar integration. Rebranded from MLSNext to YSS.

3. Phase 3: Advanced Features & Analytics (Mar 2 - Mar 10): Implemented standings page with on-the-fly calculations initially then proxy-based approach, built analytics dashboard with Momentum Index and Strength of Schedule metrics, added drill-in accordion for team match history, enabled rolling window ingestion with token auth, and extended support to FEST tournaments.



**Key Decisions:**

- Migrated from database-computed standings to proxy-based HTML parsing from Modular11 to reduce backend complexity and improve real-time accuracy

- Implemented rolling window ingestion with staggered weekly batches rather than single-pass to handle continuous tournament data and reduce API load

- Chose React Router-based multi-page architecture with standalone API Functions to enable scalability and independent feature development

- Designed full field-update upserts in match service rather than partial updates to ensure data consistency across ingestion cycles



**Primary Files:** `YSS.Ingestion/Services/ScheduleParser.cs`, `YSS.Functions/Triggers/GetStandings.cs`, `YSS.Web/src/pages/AnalyticsPage.tsx`, `YSS.Data/AppDbContext.cs`, `YSS.Functions/Triggers/GetAnalytics.cs`
