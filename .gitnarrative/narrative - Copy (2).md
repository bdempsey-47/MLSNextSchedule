### Youth Soccer Schedules Platform (YSS) — Full-Stack Development

**Complexity:** high | **Commits:** 47 | **Period:** 2026-02-26 to 2026-03-10

A comprehensive youth soccer scheduling and analytics platform built from scratch over two weeks, featuring real-time match ingestion from Modular11 API, a React-based web UI with standings and analytics pages, and token-based Azure SQL authentication. The project evolved from initial MLSNext prototype through a complete rebrand to YSS, implementing seasonal filtering, team logos, GPS-enabled venues, standings calculations, momentum-based analytics, and support for multiple tournament formats (MLS Next, Academy, FEST).


**Phases:**

1. Phase 1 (Feb 26-27): Foundation & Data Layer — Project setup, HTML parsing from Modular11 API, team-associated score parsing, database schema with migrations, and initial data population with live API data.

2. Phase 2 (Feb 27-28): Frontend Core & UI Refinement — React app scaffolding, seasonal filtering (Fall 2025/Spring 2026), URL bookmarking, modern sports-league aesthetic redesign, team logos from CloudFront CDN, and interactive badge filtering.

3. Phase 3 (Feb 28-Mar 5): Rebranding & Advanced Features — MLSNext→YSS rebrand with complete folder restructuring, rolling-window ingestion with in-memory caching, league selector, routing architecture, and Standings page with on-the-fly league table calculations via Modular11 HTML proxy.

4. Phase 4 (Mar 5-10): Analytics & Tournament Expansion — Analytics page with Momentum Index (last 8 games), Strength of Schedule calculations, Bayesian momentum weighting, GF/GA/GD metrics, team match drill-in accordions, and FEST tournament ingestion support.



**Key Decisions:**

- Adopted in-memory caching with rolling 7-day window ingestion instead of full database wipes to reduce API load and improve performance for large datasets.

- Switched from custom standings computation to Modular11 HTML proxy with gzip decompression to leverage authoritative league data and reduce calculation complexity.

- Designed modular Azure Functions (GetMatches, GetTeams, GetStandings, GetAnalytics) to decouple frontend features from backend computation, enabling independent scaling.

- Implemented Bayesian momentum weighting in analytics to balance recent performance with historical context, avoiding over-weighting outlier results.
