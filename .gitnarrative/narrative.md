## Project Overview

Youth Soccer Schedules (YSS) is a full-stack web platform for ingesting, analyzing, and displaying youth soccer match schedules and standings sourced from the Modular11 API. The backend is built on .NET Azure Functions with Entity Framework Core against Azure SQL, while the frontend is a React/TypeScript SPA. Data flows from scheduled ingestion pipelines through an upsert service into a shared database, then surfaces via HTTP-triggered Azure Functions to the React frontend.

The project originated as "MLSNext" before a full rebrand to YSS (evidenced by legacy `MLSNext.*` folder paths still present alongside `YSS.*` ones). Current state is a functioning multi-page application with Schedules, Standings, and Analytics pages, supporting both Academy (MLS Next) and FEST tournament competitions through unified schema.

## Key Features & Milestones

### Core Ingestion Pipeline & Data Model (Feb 26–Mar 10, 2026)

`ScheduleParser` extracts match data from Modular11's HTML API responses. Matches are upserted via `MatchUpsertService` with duplicate detection and team name normalization. The `IngestionOrchestrator` drives rolling-window, paginated fetches across staggered 3-batch weekly runs to stay within API rate limits, terminating pagination when no *new* matches are found (not simply on empty pages — a subtle but important distinction).

Two separate ingestion paths exist: Academy/MLS Next and FEST tournaments. Both write to a unified schema, enabling cross-competition queries without breaking changes. A venue `Location` field fallback was added to handle cases where primary venue data is absent in FEST payloads.

**Key decisions:** Pagination stops on absence of new matches rather than empty result sets. FEST and Academy share schema but have distinct parsers. Team name whitespace is trimmed at both ingestion and display layers.

**Primary files:** `YSS.Ingestion/Services/ScheduleParser.cs`, `YSS.Ingestion/Services/MatchUpsertService.cs`, `YSS.Ingestion/Services/IngestionOrchestrator.cs`, `YSS.Data/AppDbContext.cs`

### Standings Page & Modular11 HTML Proxy (Mar 5–16, 2026)

Rather than computing standings from stored match data, the backend proxies Modular11's own standings HTML directly. This reduces backend complexity and guarantees consistency with the official source. `GetStandings.cs` fetches and returns this proxied content, with fixes applied for division name uniqueness (full division name, not region name), U19 as a distinct age group separate from U18, and correct `UID_gender`/`list_type` parameter mapping.

The Standings page (`StandingsPage.tsx`) went through significant iteration: single-select program mode, region alphabetization, accordion drill-down with PascalCase→camelCase normalization, region- and league-based match filtering, and conditional hiding of redundant Homegrown metric columns (WPM, GDPM, GPM).

**Primary files:** `YSS.Functions/Triggers/GetStandings.cs`, `YSS.Web/src/pages/StandingsPage.tsx`, `YSS.Web/src/pages/StandingsPage.css`

## Traps & Gotchas

- **Team name trimming at both layers** — Whitespace is stripped in both `MatchUpsertService` (ingestion) and `StandingsPage.tsx` (display) defensively, because upstream Modular11 data has inconsistent leading whitespace. Removing either layer will cause mismatched team lookups or display inconsistencies. (`YSS.Ingestion/Services/MatchUpsertService.cs`, `YSS.Web/src/pages/StandingsPage.tsx`)

- **Pagination terminates on no new matches, not empty pages** — The `IngestionOrchestrator` stops fetching when a page returns no *new* matches, not when a page is empty. Modular11 can return pages with previously-seen records at the boundary. Reverting to an empty-page check will cause premature termination or infinite loops. (`YSS.Ingestion/Services/IngestionOrchestrator.cs`)

- **Standings are proxied HTML, not computed** — `GetStandings.cs` does not derive standings from the local database; it proxies Modular11's HTML. Any attempt to replace this with computed standings will diverge from official results and lose Modular11-specific tie-breaking logic. (`YSS.Functions/Triggers/GetStandings.cs`)

- **U19 is a separate age group from U18** — Modular11 treats U19 as a distinct category, not an alias for U18/U19. The age group map in `GetStandings.cs` explicitly includes U19 as its own key. Merging or aliasing it to U18 will break standings retrieval for U19 divisions. (`YSS.Functions/Triggers/GetStandings.cs`)

- **MLSNext.\* folders are legacy, not dead code** — The hotspot data shows active commits on `MLSNext.Web/src/components/MatchCard.css`, `MLSNext.Web/src/App.tsx`, and `MLSNext.Functions/Triggers/GetMatches.cs`. These are the pre-rebrand paths and may still be the live deployment artifacts rather than abandoned code, depending on how far the rename was applied.

- **Homegrown columns hidden via visibility toggle, not removal** — WPM, GDPM, and GPM columns are conditionally hidden in the Homegrown standings view rather than stripped from the data model, so they remain available if display requirements change. Removing them from the data pipeline would silently break the toggle. (`YSS.Web/src/pages/StandingsPage.tsx`)
